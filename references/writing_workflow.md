# Writing Workflow — ZJE 本科毕业论文

本文件是 ZJE 本科毕业论文的写作与迁移工作流指南。Script 处理 Word/XML 执行；本工作流处理问诊、中文迁移写作、模板填写和资产整理。

## 核心原则

**先问清楚，再写。** ZJE 的核心任务是"英文论文 → 中文迁移"，不是从零写作。所有决策都要落到 `00_project/project_manifest.md` 和 `00_project/thesis_master_index.md`，不要只停留在对话里。

---

## 一、问诊问题

每个新项目开始时，先确认以下事实。

### 英文论文元数据
- 英文论文题目是什么？
- 导师姓名和职称？
- IBMS10008 课程的 Abstract 是否已定稿？
- 英文论文各章节（Introduction / Methods / Results / Discussion / Conclusion）是否已完成？
- 英文论文是否有可用的 Word 文件或 PDF？

### 中文论文目标
- 中文题目是否已翻译？还是需要辅助翻译？
- 第一部分正文（论文主体）需要迁移哪些章节？
- 第二部分（文献综述、开题报告）内容来源：来自哪份 Preview 课程材料？
- 外文翻译打算选取哪篇论文？（必须是非综述、非会议论文的学术研究论文）

### 当前进度（多项选择）
- [ ] IBMS10008 英文论文已完成
- [ ] IBMS10008 英文论文进行中（哪个章节未完成？）
- [ ] Project Preview 文献综述已完成
- [ ] Project Preview 开题报告已完成
- [ ] 附件表格已部分填写
- [ ] 查重已通过

### 已有资产
- 英文论文 Word/PDF 路径
- Preview 材料路径（PDF 或 Word）
- 参考文献列表（.bib / JSON / 手写列表）
- 图片、表格、代码、数据文件
- Dissertation / Preview / Oral Presentation 的 feedback（用于填入附件表格）

### 真实性边界
- 是否允许翻译时适当意译而非字面翻译？
- 参考文献列表是否已有？还是需要辅助检索？
- 图表是从英文论文直接迁移，还是需要重新制作？

## 二、写作模式详解

### 模式A：英文论文 → 中文论文迁移（核心模式）

这是 ZJE 学生最重要的写作任务。英文论文的每个部分如何迁移：

#### 迁移对应关系

| 英文论文部分 | 迁移目标 | 说明 |
|------------|---------|------|
| Abstract | 中文摘要（300–600字） | 需翻译并压缩 |
| Abstract | 英文摘要（Abstract） | 直接保留英文原文 |
| Acknowledgement | 致谢 | 翻译为中文，中英文均可 |
| Introduction | 第1章 绪论 | 翻译后填入附件8模板 |
| Methods | 第2章（方法章节） | 翻译后填入附件8模板 |
| Results | 第3章（结果章节） | 翻译后填入附件8模板 |
| Discussion | 第4章（讨论章节） | 翻译后填入附件8模板 |
| Conclusion | 第5章 结论 | 翻译后填入附件8模板 |
| References | 参考文献 | 用于 BB 电子版和纸质版 |
| 作者自述 | 作者简历 | 用于 BB 电子版和纸质版 |

#### 中文摘要写作规范
- 字数：300–600字
- 内容：研究目的、方法、结果和结论，重点是结果和结论
- 关键词：3–8个，体现论文特色
- 不使用图、表、化学结构式、非公知公用的符号和术语

#### 第一部分正文格式（附件8模板）
- 中文字体：仿宋小四号，1.5倍行距
- 英文字体：Times New Roman 小四号，1.5倍行距
- 章节标题：仿宋三号加黑（章）、小三号仿宋加黑（节）、四号仿宋加黑（子节）
- 页眉：奇数页→论文名（居右），偶数页→"浙江大学本科生毕业论文（设计）"（居左）
- 页脚：阿拉伯数字居中，连续编码

#### 章节标题翻译
英文论文的章节标题直接翻译为中文，填入附件8模板的各章位置。附件2中的第二部分标题仅供参考，内容不必与英文论文相同。

#### 第二部分不翻译
Project Preview 的文献综述和开题报告内容：**直接使用英文原文**，填入附件2模板，不需要翻译。

---

### 模式B：填写 Part 2 模板（第二部分）

#### 文献综述（附件2模板·英文原文）
结构：
1. 背景介绍
2. 国内外研究现状（研究方向及进展、存在问题）
3. 研究展望
4. 参考文献

#### 开题报告（附件2模板·英文原文）
结构：
1. 问题提出的背景（背景介绍、本研究的意义和目的）
2. 论文的主要内容和技术路线（主要研究内容、技术路线、可行性分析）
3. 研究计划进度安排及预期目标
4. 参考文献

**注意**：如果 Preview 内容与 Dissertation 主题存在偏差，允许对 Preview 进行适当修改，使其与 Dissertation 主题保持一致。

#### 外文翻译（附件2模板·中文）
- 选取一篇与毕设课题相关的学术研究论文（非综述、非会议论文）
- 格式：中文翻译，仿宋小四号，1.5倍行距
- 外文原文附在最后（文字粘贴或截图均可）

---

### 模式C：填写附件表格

#### 附件1·任务书
- 题目：论文的中文题目
- 进度安排及任务要求：从 Project Description 或 "Guidance and Agreement for ZJE Year 4 External Projects Supervisors" 文件中摘取英文原文填入
- 起讫日期：2026年1月12日至2026年4月12日
- 导师签名：电子签或手签
- 审核意见：**不填**

#### 附件2·指导教师具体要求
- 题目：论文的中文题目
- 指导教师对文献综述和开题报告的具体要求：从 Project Description 摘取英文原文填入，或从 Preview 课程 guidance 中摘取，或联系导师填写（中英文均可）
- 导师签名：电子签或手签
- 日期：2026年1月12日

#### 附件3·文献综述和开题报告考核表
- 将 Project Preview 的 feedback（BB上 Preview 课程的 submission dropbox 里）**原文粘贴**，不需要翻译
- 成绩和签名：**不填**

#### 附件9·专家评阅意见表
- 将 Dissertation 的 feedback **原文粘贴**，不需要翻译
- 同意答辩：填"√"
- 评阅人信息：**不填**

#### 附件11·毕业论文考核表（附件11）
- 导师评语：联系导师写一段评语（中英文均可）
- 将 Oral Presentation 的 feedback（BB上 Project 课程的 submission dropbox 里）**原文粘贴**，不需要翻译
- 成绩和答辩小组签名：**不填**

#### 附件10·现场答辩记录表
- 答辩当天现场填写，填写后到办公室领取

---

### 模式D：参考文献与资产管理

#### 参考文献格式
使用 GB/T 7714-2015 标准，顺序编码制或著者-出版年制，**全文必须统一**。

#### ZJE 特有的引用说明
- 第一部分正文中的引用：按 GB/T 7714 执行
- 第二部分文献综述中的引用：英文原文中的引用格式保留
- 外文翻译：翻译文中涉及的引用须一并翻译，格式按 GB/T 7714

#### 图表管理
- 图题：五号宋体加粗
- 表题：五号宋体加粗
- 表格：国际通行的三线表（顶线、底线、表头底线）
- 表格中文字体：五号宋体，单倍行距

---

### 模式E：三版本 Word 输出

#### 查重版
- 包含：大封面、致谢、目录、中文摘要、英文摘要、正文（不含参考文献和简历）
- 格式：Office Word 或 WPS Word，**禁止 PDF**
- 操作：完成第一部分1–8后，另存为查重专用版本，删除承诺书、参考文献、个人简历

#### BB 提交版
- 包含：除任务书、考核表（附件11）、指导教师具体要求、考核表（附件3）、查重报告、专家评阅意见、答辩记录表之外的全部内容
- 承诺书电子版可不签名

#### 纸质存档版
- 包含：全部内容（按 `中文论文材料顺序-全局索引.xlsx` 的顺序排列）
- 附件表格先打印后签字

## 三、ZJE 论文大纲框架

### 第一部分：毕业论文（设计）正文

**结构由英文论文决定**，英文论文各章节标题翻译后即为中文章节标题。

默认参考结构（根据英文论文内容调整）：
1. 第1章 绪论（Introduction）
2. 第2章 [方法章节名]（Methods）
3. 第3章 [结果章节名]（Results）
4. 第4章 [讨论章节名]（Discussion）
5. 第5章 结论（Conclusion）

### 第二部分：文献综述和开题报告

**结构来自 Project Preview 课程材料**，英文原文填入模板：
1. 文献综述
   - 1 背景介绍
   - 2 国内外研究现状（2.1 研究方向及进展 / 2.2 存在问题）
   - 3 研究展望
   - 4 参考文献
2. 开题报告
   - 1 问题提出的背景（1.1 背景介绍 / 1.2 本研究的意义和目的）
   - 2 论文的主要内容和技术路线（2.1 主要研究内容 / 2.2 技术路线 / 2.3 可行性分析）
   - 3 研究计划进度安排及预期目标（3.1 进度安排 / 3.2 预期目标）
   - 4 参考文献
3. 外文翻译
4. 外文原文

## 四、章节计划

写章节前，先确认或生成 `03_chapters/chXX_plan.md`：

```
# 第X章 标题
<!-- chapter_id: ch_X -->

## 本章来源
- 英文论文对应章节：[英文章节名]
- 翻译状态：[未翻译 / 进行中 / 已完成]

## 本章目标
本章解决什么问题。

## 二级/三级标题
确认的标题必须先记录。

## 输入资产
参考文献、代码、数据、图、表、公式、已有文字。

## 待补项
缺失材料和需要用户确认的问题。

## 格式要求
- 字体：仿宋小四号，1.5倍行距
- 英文字体：Times New Roman 小四号
- 图表标题：五号宋体加粗
```

## 五、资产整理规范

- 参考文献放在 `08_refs/`，格式按 GB/T 7714
- 代码放在 `06_code/`，用于方法描述和实现细节
- 数据放在 `07_data/`，用于数据集描述、实验设置和结果分析
- 图片放在 `04_figures/`，图题用占位符管理
- 表格放在 `05_tables/`，表题用占位符管理
- 反解析得到的图、表须查看 manifest 中的 `usable` 状态
- 所有资产在 manifest 中记录状态：已有、待补、已采用、需确认

---

## 六、术语规范

- 严禁创造不存在的学术词汇
- 术语来源：用户提供材料、参考文献、学科通用表达、用户明确命名的本文方法
- 创新模块命名：必须写成"本文设计的……"或"本文称其为……"
- 专业名词前后保持一致，不要反复换名字

---

## 七、学术语体

- 客观、准确、克制
- 避免口语化、宣传式、过度绝对化表达
- 结论必须有依据；无支撑时使用"可能""有助于"等审慎表述
- 第一章重视问题提出和研究意义；第二章重视已有方法和本文工作的位置；第三章重视方法逻辑；第四章重视实验依据；第五章重视总结和边界

---

## 八、Markdown 文件职责

- `00_project/project_manifest.md`：英文论文元数据、中文论文目标、当前进度、资产、缺失项
- `00_project/thesis_master_index.md`：两大部分大纲、章节状态、术语表、开放问题
- `00_project/decisions_log.md`：用户确认的关键决策和写作边界
- `03_chapters/chXX_plan.md`：章节计划（翻译状态、待补项、格式要求）
- `03_chapters/chXX_draft.md`：已确认的中文章节正文
